## 들어가며


이 노트는 Antonio Gulli의 _Agentic Design Patterns: A Hands-On Guide to Building Intelligent Systems_를 읽으며, 제가 이해한 내용을 제 언어로 다시 정리한 기록입니다. 번역도, 요약본도 아닙니다. 한 챕터씩 읽고  이 패턴이 무엇을 풀고, 어떻게 작동하며, 제 프로젝트에 어떻게 닿는지를 제 말로 적어 손에 남기는 것이 목적입니다.


이 책을 고른 이유는 분명합니다. 저는 DevQuest를 만들면서 모델 라우팅, 비용·캐시 최적화, 에이전트 격리 하네스 같은 것들을 이미 손으로 만들었습니다. 다만 그것들을 "패턴 언어"로 부를 줄은 몰랐습니다. 이 책은 제가 직관과 필요로 만든 것들에 이름과 구조를 붙여줍니다. Routing, Resource-Aware Optimization, Guardrails. 제가 한 일이 사실은 이런 패턴이었음을 알게 되는 과정이고, 그래야 다음엔 의도적으로 설계할 수 있습니다.


그래서 이 노트의 기준은 "얼마나 빨리 읽었나"가 아니라 "얼마나 남았나"입니다. 어줍잖게 21개를 훑느니, 한 패턴이라도 그것이 무엇을 풀려고 태어났는지, 언제 쓰고 언제 쓰지 말아야 하는지, 그 트레이드오프가 무엇인지까지 설명할 수 있을 만큼 체득하는 것을 목표로 합니다.


정리 방식은 챕터마다 같습니다. **핵심 개념(무엇을·왜)**, **메커니즘과 구조(어떻게)**, **제 프로젝트·실무와의 연결**, 그리고 **직접 손으로 확인**입니다. 읽고 끄덕인 것은 사흘이면 사라지지만, 제 문장으로 다시 쓰고 코드로 만져본 것은 남는다고 믿습니다.

## 에이전틱 시스템이란?


에이전틱 시스템은 전통적으로 고정된 스크립트를 따르는게 아닌 지속적인 **인간 감독 없이 행동할 수 있는 자율성, 능동성, 변화에 대응하는 반응성을 특성**으로 하며 근본적으로 끊임없이 목표를 향해 나아갑니다. 핵심 역량은 **도구사용**으로 외부 API, 데이터베이스 또는 서비스와 상호작용하고, 정보를 유지하는 메모리를 보유함으로서 사용자, 다른 시스템이나 연결된 다른 에이전트와도 소통할 수 있습니다.


단, 이러한 특성들을 효과적으로 구현하는 건 상당한 복잡성을 수반합니다. 어떻게 상태를 유지시킬지, 도구를 언제 어떻게 사용할지, 다른 에이전트와의 소통은 어떻게 관리할지, 의도하지 않은 결과나 오류가 발생했을 때 시스템을 어떻게 복원해야 할 지 고려해야 합니다.


전통적 시스템과의 차이를 한 장면으로 보면 이렇습니다. 고객 문의를 처리하는 시스템을 떠올려 봅니다.


|       | 전통적 시스템      | 에이전틱 시스템             |
| ----- | ------------ | -------------------- |
| 동작    | 고정된 스크립트를 따름 | 문의의 뉘앙스를 인식          |
| 지식    | 분기문에 박힌 규칙   | 지식 베이스에 접근           |
| 외부 연동 | 미리 정의된 호출만   | 주문 관리 등 다른 시스템과 상호작용 |
| 불확실성  | 처리 불가 → 에러   | 명확화 질문을 던지고 진행       |

## 에이전틱 개발에서 패턴이 중요한 이유


이러한 복잡성을 해결하기 위해 설계 패턴을 활용하면 대화 흐름 관리, 외부 기능 통합, 다중 에이전트 행동 조율 같은 작업에 대한 근본적인 해결책을 처음부터 다시 발명하는 수고를 피할 수 있습니다. 패턴은 에이전트 로직을 더 명확하고, 다른 사람들이 이해하고 유지하기 쉽게 만드는 공통 언어와 구조를 제공합니다. 이러한 검증된 접근법을 활용하면 개발 프로세스가 가속화되어, 에이전트 동작의 근본적인 메커니즘보다 애플리케이션의 고유한 측면에 집중할 수 있습니다.

> 저자는 책에서 "AI가 이렇게 빨리 변하는데 왜 금방 낡을 책을 쓰는가?"라는 질문에 정반대로 답합니다. 너무 빨리 변하기 때문에, 한 걸음 물러서서 자리를 잡아가는 근본 원칙 — RAG, 반성(Reflection), 라우팅, 메모리 — 을 짚어야 한다는 것입니다. 이 시리즈도 같은 이유에서 출발합니다.

## 시리즈 구성과 읽는 법


이 책은 21개 챕터, 한 챕터당 하나의 패턴을 다룹니다. 각 패턴은 다음 흐름으로 정리합니다.


| 단계           | 내용                |
| ------------ | ----------------- |
| 핵심 개념(무엇·왜)  | 이 패턴이 무엇이고 왜 필요한가 |
| 메커니즘·구조(어떻게) | 어떻게 작동하는가         |
| 내 프로젝트·실무 연결 | 내가 만든 것과 어디서 닿는가  |
| 직접 손으로 확인    | 코드로 만져보고 남긴다      |


순서대로 읽으면 개념이 쌓이지만, 지금 막힌 문제의 패턴으로 바로 건너뛰는 참고서로 써도 됩니다. 코드 예제는 LangChain(+LangGraph), CrewAI, Google ADK 세 가지를 "캔버스"로 번갈아 씁니다. 특정 기술에 묶이지 않고 패턴 자체를 보기 위해서입니다.
